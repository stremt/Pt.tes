The Ingredient Health Analyzer must be upgraded to produce realistic and accurate health evaluations based on food ingredients. The current system works but still needs improvements to make the scoring more reliable and scientifically consistent.

First, improve ingredient parsing and normalization.

The analyzer must extract ingredients only from the user input field. It should split ingredients primarily by commas and clean the text by removing percentages, brackets, and extra formatting. The system should normalize ingredient names so that different label formats map to the same ingredient.

Examples of normalization:
“flavor enhancer (msg)” → msg
“monosodium glutamate” → msg
“e621” → msg
“refined wheat flour” → refined flour
“whole grain oats” → oats

Duplicate ingredients must be removed so each ingredient is evaluated only once.

Second, add strict ingredient validation.

The analyzer should only run when at least two valid food ingredients are detected. If fewer than two recognized ingredients are found, the tool should display:

“No valid ingredient list detected. Please paste ingredients exactly as written on a food label.”

This prevents random text from being analyzed.

Third, implement ingredient quantity weighting.

Food labels list ingredients in descending order by quantity. Earlier ingredients should have stronger influence on the score.

Use this weighting model:

1st ingredient → weight 1.5
2nd ingredient → weight 1.3
3rd ingredient → weight 1.1
4th ingredient → weight 1.0
5th ingredient → weight 0.8
Remaining ingredients → weight 0.6

This ensures the main ingredients affect the health score more strongly.

Fourth, redesign the scoring system.

Start with a base score of 60 and adjust based on ingredient categories.

Healthy ingredients:
Whole grains → +15
Nuts → +12
Seeds → +12
Legumes → +10
Fiber-rich ingredients → +10
Natural plant foods → +8
Protein sources → +8

Moderate ingredients:
Honey → −5
Salt → −5
Vegetable oil → −6
Fruit concentrates → −6

Harmful ingredients:
Sugar → −20
High fructose corn syrup → −25
Palm oil → −15
Refined flour → −15
Maltodextrin → −15
Artificial color → −15
Artificial flavor → −10
MSG → −10
Hydrogenated oil → −25
Trans fats → −30

Clamp the final score between 0 and 100.

Fifth, improve positive ingredient detection.

Highlight beneficial ingredients such as whole grains, nuts, seeds, natural protein sources, and fiber sources. These should appear in a “Positive Ingredients” section.

Sixth, improve unknown ingredient handling.

Limit the display of unknown ingredients to a maximum of five items. Do not list dozens or hundreds of unknown ingredients.

Seventh, add combination risk detection.

Detect unhealthy ingredient combinations such as:

Sugar + palm oil → ultra processed warning
Sugar + maltodextrin → high glycemic load warning
Artificial color + artificial flavor → additive-heavy product warning

These warnings should appear in the report.

Eighth, improve the final verdict logic.

Score interpretation:

80–95 → Excellent / Very Healthy
60–79 → Healthy
40–59 → Moderate / Processed
20–39 → Unhealthy
0–19 → Highly Unhealthy / Avoid

Even very healthy foods should rarely reach a perfect score of 100 unless they contain only natural whole ingredients.

Finally, improve report clarity.

The final report should clearly show:

Total detected ingredients
Healthy ingredient count
Moderate ingredient count
Harmful ingredient count
Key warnings
Positive ingredients
Overall health score
Final verdict

The analyzer should behave like a smart food label interpreter that gives realistic health insights rather than generic scoring.
