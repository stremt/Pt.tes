Fix the live preview pagination system of the Text to PDF tool. The current implementation creates layout issues where text and tables appear misaligned in the preview and multiple empty pages are generated when the user edits content.

Important observation:
The exported PDF renders correctly, but the live preview incorrectly generates extra pages and layout breaks.

This indicates the preview pagination logic is recalculating pages incorrectly.

Implement the following fixes.

1. **Reset Preview Before Rendering**

Every time the preview updates, completely clear the preview container before rebuilding pages.

previewContainer.innerHTML = "";

This prevents duplicate or empty pages from accumulating after edits.

2. **Single Source Rendering**

Ensure the preview and PDF generator use the same HTML content container.

Rendering pipeline must be:

Markdown/Text → HTML → Preview → PDF

Do not create separate rendering structures for preview and PDF.

3. **Correct Page Height Calculation**

Define fixed A4 preview dimensions:

.page {
width: 794px;
min-height: 1123px;
}

Use content height measurement instead of manual page counting:

const contentHeight = content.scrollHeight;

Avoid creating new pages unless content exceeds the page height.

4. **Prevent Duplicate Page Creation**

Before adding a new page, verify the previous page is actually full.

if(currentPageHeight < pageHeight) {
continue writing content
}

Only create a new page when the content truly exceeds page height.

5. **Prevent Empty Page Creation**

Do not generate pages that contain no content.

Add validation:

if(pageContent.trim() === "") {
do not append page
}

6. **Fix Table and Text Layout**

Ensure tables and paragraphs cannot break across pages incorrectly.

Apply CSS:

table {
page-break-inside: avoid;
}

p, h1, h2, h3 {
page-break-inside: avoid;
}

7. **Avoid Page Rebuild on Every Keystroke**

Currently the preview likely rebuilds on every character typed.

Add debounce to reduce excessive re-rendering:

const debounceRender = debounce(renderPreview, 300);

This prevents continuous page creation while typing.

8. **Stable Layout Rendering**

Ensure layout uses CSS flow instead of manual DOM slicing.

Allow the browser layout engine to control element positioning and only calculate page breaks afterward.

9. **Final Goal**

The preview should behave exactly like the generated PDF:

* no empty pages
* no duplicate pages
* tables aligned
* text layout correct
* page count stable
* editing should not create new pages unnecessarily
