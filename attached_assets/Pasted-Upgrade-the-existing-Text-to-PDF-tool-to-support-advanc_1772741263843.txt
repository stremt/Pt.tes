Upgrade the existing **Text to PDF tool** to support advanced content rendering while keeping the application **fully static * with no backend or server-side processing.

The current tool correctly converts plain text and tables into PDF, but the next version must support advanced formatting such as **LaTeX math equations, syntax-highlighted code blocks, and embedded images positioned exactly where the user places them in the text**.

The upgraded system should include the following improvements:

1. **LaTeX Math Rendering (KaTeX)**

* Integrate **KaTeX** as a client-side library.
* Automatically detect LaTeX math expressions inside:

  * `$...$` for inline math
  * `$$...$$` for block equations
* Render equations before generating the PDF.
* Ensure proper support for:

  * fractions
  * square roots
  * integrals
  * summations
  * matrices
  * superscripts and subscripts
* Prevent line breaking inside equations.

Example supported input:

$$
\int (4x^3 - 6x + 2) dx
$$

$$
x = \frac{-b \pm \sqrt{b^2 - 4ac}}{2a}
$$

2. **Syntax Highlighted Code Blocks**

* Add syntax highlighting using **Prism.js or Highlight.js**.
* Support common programming languages:

  * JavaScript
  * Python
  * HTML
  * CSS
  * JSON
  * Bash
* Code blocks should follow Markdown style:

```javascript
function generatePDF(text){
  return convertToPDF(text);
}
```

* Preserve indentation and formatting in the final PDF.
* Use a monospace font for code blocks.

3. **Image Embedding Support**
   The tool must allow users to insert images inside the text and ensure the image appears in the **exact same position inside the generated PDF**.

Support the following methods:

A) Markdown image format:

`![Image Description](image-url)`

B) HTML image tag:

`<img src="image-url" width="300">`

C) Local image upload:

* Add a file input so users can upload an image.
* Automatically insert the image into the editor at the cursor position.

Requirements for image rendering:

* Maintain aspect ratio.
* Support resizing.
* Prevent images from overflowing page margins.
* Automatically move large images to the next page if needed.

4. **Improved PDF Rendering Engine**

Replace the current simple PDF generator with a stable static-compatible pipeline:

HTML → Canvas → PDF

Use:

* **html2pdf.js**
* **html2canvas**
* **jsPDF**

Ensure:

* Layout preservation
* Proper page breaks
* Accurate spacing
* Multi-page document support

5. **Structured Rendering Pipeline**

Before generating the PDF:

Step 1: Parse Markdown (use `marked.js`)
Step 2: Render LaTeX equations (KaTeX)
Step 3: Apply syntax highlighting (Prism.js or Highlight.js)
Step 4: Render images
Step 5: Convert final HTML to PDF

6. **Editor Improvements**

Upgrade the input area into a rich editor with:

* Live preview panel
* Markdown rendering
* Equation preview
* Image preview
* Code syntax highlighting preview

7. **Typography and Layout**

Improve PDF readability:

* Use a professional font stack
* Maintain consistent margins
* Prevent orphan lines
* Ensure page breaks do not split tables or code blocks

8. **Static Architecture Requirement**

The tool must remain:

* Fully client-side
* Compatible with static hosting platforms
* Lightweight and fast
* No backend dependencies

All libraries must load via CDN or local static files.

9. **Final Expected Result**

The upgraded tool must be capable of converting documents containing:

* paragraphs
* markdown formatting
* large tables
* LaTeX math equations
* syntax highlighted code blocks
* embedded images
* multilingual Unicode text
* multi-page documents

into a **clean, well-formatted PDF with identical visual structure**.
