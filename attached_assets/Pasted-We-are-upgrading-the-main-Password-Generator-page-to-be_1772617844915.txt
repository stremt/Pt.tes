Title:

Security Transparency: How Pixocraft Generates Passwords

Intro paragraph:

Pixocraft Tools uses modern browser cryptography to generate truly unpredictable passwords. Unlike many basic generators that rely on predictable random functions, this tool uses the Web Crypto API to produce cryptographically secure randomness suitable for protecting sensitive credentials.

Subsection:

Cryptographically Secure Randomness

Explain:

Passwords are generated using the browser's native cryptographic random number generator:

window.crypto.getRandomValues()

This API produces CSPRNG values (Cryptographically Secure Pseudo-Random Numbers) designed for security-critical operations.

These values are used to construct password characters with maximum entropy and unpredictability.

Subsection:

Why We Never Use Math.random()

Explain briefly:

Traditional random functions such as:

Math.random()

timestamp seeded randomness

predictable algorithms

are not suitable for security applications because they can produce patterns that attackers may exploit.

Pixocraft avoids these methods entirely.

Subsection:

Client-Side Password Generation

Explain clearly:

All password generation occurs inside your browser.

This guarantees:

• No password transmission
• No server-side logging
• No password storage
• No external API calls

Your generated password never leaves your device.

What Does 105 Bits of Entropy Actually Mean?

Entropy measures how unpredictable a password is.

A typical 16-character password generated by this tool contains approximately 105 bits of entropy.
