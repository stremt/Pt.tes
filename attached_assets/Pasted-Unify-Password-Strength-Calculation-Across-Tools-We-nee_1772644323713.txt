Unify Password Strength Calculation Across Tools

We need to ensure that Password Generator and Password Strength Checker use the exact same strength calculation logic.

Currently users may see different strength results, which creates confusion.

Goal:

A password generated in the Pixocraft Password Generator should show exactly the same entropy score and strength rating when tested in the Password Strength Checker.

Both tools must share the same entropy calculation engine.

1️⃣ CREATE SHARED PASSWORD SECURITY MODULE

Create a reusable module:

passwordSecurity.ts

This module must contain:

• entropy calculation
• character set detection
• strength level classification
• crack time estimation

Both tools must import this module.

2️⃣ ENTROPY CALCULATION FORMULA

Use this formula:

Entropy = L × log2(R)

Where:

L = password length
R = character set size

Character set detection rules:

Lowercase → +26
Uppercase → +26
Numbers → +10
Symbols → +33

Example:

If password contains all sets:

R ≈ 95

Example entropy:

16 × log2(95) ≈ 105 bits
3️⃣ CRACK TIME ESTIMATION

Estimate brute-force cracking time using:

Total combinations = 2^entropy

Assume attack speed:

1e12 guesses per second

Calculate approximate time.

Convert into readable format:

seconds
minutes
hours
years
longer than age of universe

4️⃣ STRENGTH CLASSIFICATION

Use the same thresholds everywhere.

Strength levels:

0–40 bits → Weak
40–70 bits → Moderate
70–100 bits → Strong
100+ bits → Very Strong

Both tools must display identical strength labels.

5️⃣ GENERATOR INTEGRATION

When generator creates a password:

Call:

calculatePasswordSecurity(password)

Return:

entropy
strengthLabel
crackTime
securityScore

Display those results immediately.

6️⃣ CHECKER INTEGRATION

Password Strength Checker must call the same function:

calculatePasswordSecurity(password)

No custom logic allowed.
