The Ingredient Health Analyzer tool currently has several critical logic problems that must be fixed so that the analysis becomes accurate and reliable.

The current issue is that the system is incorrectly detecting random text from the page as ingredients, producing hundreds of “unknown ingredients,” and generating a health score even when the input is not a real ingredient list. The analyzer must be redesigned to properly parse ingredient lists and validate the input.

First, fix the ingredient parsing system.

The analyzer must only read the text entered inside the ingredient input field. It must not analyze any text from the page layout, headers, navigation, or other UI elements.

The parser should clean the input text and extract ingredients properly. Ingredients should be separated primarily by commas, but the system should also handle brackets and percentages commonly found in food labels.

Example input:
Sugar, Palm Oil, Cocoa Powder, Flavor Enhancer (MSG), Salt

The parser should extract and normalize these ingredients as:
sugar
palm oil
cocoa powder
msg
salt

Remove duplicate ingredients so that each ingredient is analyzed only once.

Next, implement ingredient validation.

If the analyzer detects fewer than two valid food ingredients, the system should not perform scoring. Instead, it should show a message such as:

“No valid ingredient list detected. Please paste ingredients exactly as shown on a food label.”

This prevents random text like “hello world” from being analyzed.

Next, implement quantity weighting.

In most food labels, ingredients are listed in descending order by quantity. The analyzer should account for this by assigning higher influence to earlier ingredients.

Suggested weighting model:

1st ingredient → weight 1.5
2nd ingredient → weight 1.3
3rd ingredient → weight 1.1
4th ingredient → weight 1.0
5th ingredient → weight 0.8
Remaining ingredients → weight 0.6

This makes the health score more realistic.

Next, improve the scoring system.

Start with a base score of 60 and adjust the score based on ingredient health impact.

Example scoring guidelines:

Healthy ingredients:
Whole grains → +15
Nuts → +12
Seeds → +12
Fiber sources → +10
Natural plant ingredients → +8

Moderate ingredients:
Honey → −5
Salt → −5
Vegetable oil → −5

Harmful ingredients:
Sugar → −20
High fructose corn syrup → −25
Palm oil → −15
Refined wheat flour → −15
Artificial color → −15
Artificial flavor → −10
MSG → −10
Hydrogenated oils → −25
Trans fats → −30

Clamp the final score between 0 and 100.

Next, fix the unknown ingredient system.

Currently the tool produces dozens or hundreds of unknown ingredients. This should be limited.

Only show a maximum of five unknown ingredients in the report. Ignore the rest.

Next, implement ingredient normalization rules so that different naming formats are recognized correctly.

Examples:
“flavor enhancer (msg)” → msg
“monosodium glutamate” → msg
“e621” → msg
“refined wheat flour” → refined flour

This ensures the analyzer recognizes ingredients even when labels use different formats.

Next, add combination warnings.

The analyzer should detect risky ingredient combinations such as:

Sugar + palm oil → ultra processed food warning
Sugar + maltodextrin → high glycemic load
Artificial flavor + artificial color → additive heavy product

Next, improve the final verdict logic.

Score ranges should be interpreted as:

80–100 → Excellent / Very Healthy
60–79 → Healthy
40–59 → Moderate / Processed
20–39 → Unhealthy
0–19 → Highly Unhealthy / Avoid

Finally, improve the report clarity.

The report should show:

Detected Ingredients count
Healthy ingredient count
Moderate ingredient count
Harmful ingredient count
Top warnings
Positive ingredients
Final health score
Final verdict

The analyzer must behave reliably and must not generate analysis when the input text is not a valid ingredient list.
